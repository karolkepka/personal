Essence: governed CDC from on-prem Oracle
to multi-account Redshift — with safe lower envs

General scope — State policy bank in an active war: exposure to air and missile threats, fragile physical and network continuity, and elevated operational risk (security, compliance, and service delivery). The programme assumes contested operations, not a calm “lift-and-shift” only.

Customer vision (high-level SoW) for JSC Ukreximbank: establish a governed AWS Organization (logs, dev, test, UAT, prod, backups, tools), run DMS CDC from on-prem Oracle to Redshift, protect lower environments with masked/anonymized data, and release IaC/CaC via CI/CD with UAT/PROD approvals.

Oracle (on-prem) DMS CDC Redshift estates Terraform · CI/CD
  • Target data path — CDC pipes from Oracle SRC into account-scoped Redshift (PROD DWH; parallel feeds to UAT/DEV). Non-prod consumers use masked or anonymised schemas so teams can build without exposing sensitive production detail.
  • Delivery model — Multi-account CI/CD (CodeCommit, CodePipeline, CodeBuild), Secrets Manager, manual gates for UAT/PROD, Terraform landing zone; Fargate + EventBridge for transforms, scaling to MWAA as ETL breadth grows (15+ DMS tasks, 70+ workstreams).
